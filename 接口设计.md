# 接口设计文档

## 1. 账户与认证模块

### 1.1 注册账号

**接口功能**：读者注册账号，以便使用图书馆系统的在线服务

**请求URL**：`/api/register`

**请求方法**：`POST`

**请求参数**：
```json
{
  "name": "string",
  "email": "string",
  "password": "string",
  "studentId": "string"
}
```

| 参数名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| name | String | 是 | 姓名 |
| email | String | 是 | 邮箱（作为登录账号） |
| password | String | 是 | 密码 |
| studentId | String | 否 | 学号（学生可填） |

**响应格式**：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "userId": "string",
    "name": "string",
    "email": "string",
    "role": "STUDENT"
  }
}
```

**错误响应示例**：
```json
{
  "code": 400,
  "message": "该邮箱已被注册",
  "data": null
}
```

**主要错误码**：
- 200: 注册成功
- 400: 参数错误或邮箱已被注册

---

### 1.2 登陆系统

**接口功能**：用户登录系统，根据角色返回不同的登录结果

**请求URL**：`/api/login`

**请求方法**：`POST`

**请求参数**：

```json
{
  "userName": "string",
  "password": "string"
}
```

| 参数名   | 类型   | 是否必填 | 说明                 |
| -------- | ------ | -------- | -------------------- |
| userName | String | 是       | 邮箱（作为登陆账号） |
| password | String | 是       | 密码                 |

**响应格式**：

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "string",
    "userId": 1,
    "userName": "string",
    "role": "user"
  }
}
```

**错误响应示例**：

```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": null
}
```

**主要错误码**：

- 200: 登录成功
- 401: 用户名或密码错误
- 400: 参数错误



---

## 2. 图书检索模块

### 2.1 查看所有图书

**接口功能**：分页查看馆藏图书列表，支持基础浏览

**请求URL**：`/api/books`

**请求方法**：`GET`

**请求参数**：

| 参数名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| page | Number | 否 | 页码，从 1 开始，默认 1 |
| size | Number | 否 | 每页数量，默认 10 |

**响应格式**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 100,
    "page": 1,
    "size": 10,
    "list": [
      {
        "id": "string",
        "title": "string",
        "author": "string",
        "isbn": "string",
        "genre": "string",
        "cover": "string",
        "available": true,
        "availableCopies": 3,
        "createdAt": "YYYY-MM-DD HH:mm:ss"
      }
    ]
  }
}
```

**主要错误码**：
- 200: 成功
- 400: 参数错误

---

### 2.2 根据条件查找图书（按书名 / 作者搜索）

**接口功能**：按书名或作者搜索图书，快速找到感兴趣的书或某作者作品

**请求URL**：`/api/books/search`

**请求方法**：`GET`

**请求参数**：

| 参数名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| keyword | String | 是 | 搜索关键词（匹配书名或作者） |
| type | String | 否 | 搜索类型：`title`（书名）/ `author`（作者），不传则同时匹配书名与作者 |
| page | Number | 否 | 页码，默认 1 |
| size | Number | 否 | 每页数量，默认 10 |

**响应格式**：与 2.1 一致，`data` 为分页结构，`list` 为图书简要信息数组。

**主要错误码**：
- 200: 成功
- 400: 缺少 keyword 或参数错误

---

### 2.3 搜索筛选（按图书分类 / 语言）与排序（按日期 / 评分）

**接口功能**：在检索结果中按分类、语言筛选，并按上架日期或评分排序，更快找到合适的书

**请求URL**：`/api/books`

**请求方法**：`GET`

**请求参数**：

| 参数名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| keyword | String | 否 | 搜索关键词（可选，与 2.2 组合使用） |
| genre | String | 否 | 图书分类筛选：`Technology` / `Fiction` / `Science` / `History` / `Management` |
| language | String | 否 | 语言筛选：`Chinese` / `English` / `Others` |
| sortBy | String | 否 | 排序字段：`createdAt`（上架日期）/ `rating`（评分） |
| sortOrder | String | 否 | 排序方向：`asc` / `desc`，默认 `desc` |
| page | Number | 否 | 页码，默认 1 |
| size | Number | 否 | 每页数量，默认 10 |

**响应格式**：与 2.1 一致；若按评分排序，`list` 中每项可包含 `averageRating` 字段。

**主要错误码**：
- 200: 成功
- 400: 参数错误

### 4.7 馆员条形码查询

**接口功能**：馆员通过扫码枪或手动输入条形码、ISBN、图书 ID 查询图书，用于借书前确认馆藏状态

**请求方法**：`GET`

**请求参数**：路径参数 `code` 为条形码内容，支持直接传 ISBN、图书 ID，或带前缀的 `isbn:9780132350884`、`book:{bookId}`

**响应格式**：
```json
{
  "code": 200,
  "message": "Operation successful",
  "data": {
    "id": "string",
    "title": "string",
    "author": "string",
    "isbn": "string",
    "genre": "Technology",
    "language": "English",
    "shelfLocation": "TECH-001",
    "available": true,
    "availableCopies": 3,
    "barcode": "9780132350884",
    "barcodeType": "ISBN"
  }
}
```

**主要错误码**：
- 200: 查询成功
- 400: 条形码为空或格式无效
- 401: 未登录
- 403: 非馆员或管理员
- 404: 未匹配到图书

### 4.8 馆员 Fine Dashboard

**接口功能**：馆员查看仪表盘统计，包括当前在馆册数、当前借出册数、逾期册数，以及今日应收罚款明细

**请求URL**：`/api/librarian/fine-dashboard`

**请求方法**：`GET`

**响应格式**：
```json
{
  "code": 200,
  "message": "Operation successful",
  "data": {
    "booksInLibrary": 39,
    "checkedOutBooks": 5,
    "overdueBooks": 2,
    "fineDueToday": 10,
    "fineItemCount": 2,
    "fineItems": [
      {
        "loanId": "string",
        "userId": "string",
        "userName": "Student One",
        "userEmail": "student1@library.com",
        "studentId": "S10001",
        "bookId": "string",
        "bookTitle": "Clean Code",
        "isbn": "9780132350884",
        "dueDate": "YYYY-MM-DD HH:mm:ss",
        "returnDate": "YYYY-MM-DD HH:mm:ss",
        "fineAmount": 5,
        "status": "Returned"
      }
    ],
    "generatedAt": "YYYY-MM-DD HH:mm:ss"
  }
}
```

**统计口径**：
- `booksInLibrary`：所有图书 `availableCopies` 总和
- `checkedOutBooks`：状态为 `Borrowing` 或 `Overdue` 的借阅记录数
- `fineDueToday`：所有 `fineAmount > 0` 且未支付、未免除的罚款总额
- `fineItems`：今日需要馆员收取的未支付罚款清单

**主要错误码**：
- 200: 查询成功
- 401: 未登录
- 403: 非馆员或管理员

---

### 4.9 馆员支付宝收取罚款

**接口功能**：馆员通过支付宝收取读者逾期罚款。当前实现为模拟支付宝支付流程：接口调用成功后将该借阅记录的罚款标记为已支付，并从 Fine Dashboard 待收款清单中移除

**请求URL**：`/api/librarian/loans/{id}/pay-fine`

**请求方法**：`POST`

**请求参数**：路径参数 `id` 为产生罚款的借阅记录 ID

```json
{
  "amount": 5,
  "method": "ALIPAY",
  "authCode": "281234567890123456"
}
```

| 参数名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| amount | Number | 否 | 支付金额；传入时必须等于应缴罚款 |
| method | String | 否 | 支付方式，前端传 `ALIPAY` |
| authCode | String | 是 | 模拟支付宝付款码，以 `28` 开头 |

**响应格式**：
```json
{
  "code": 200,
  "message": "Fine paid successfully via Alipay",
  "data": {
    "loanId": "string",
    "userId": "string",
    "userName": "Student One",
    "bookId": "string",
    "bookTitle": "Clean Code",
    "fineAmount": 5,
    "finePaid": true,
    "paymentMethod": "ALIPAY",
    "outTradeNo": "FINE-loanId",
    "alipayTradeNo": "ALI1763970000000123456"
  }
}
```

**主要错误码**：
- 200: 支付成功
- 400: 该借阅记录没有可支付罚款、罚款已支付/免除、或金额不匹配
- 401: 未登录
- 403: 非馆员或管理员
- 404: 借阅记录不存在

## 10. 通用说明

### 10.1 统一响应格式

所有接口统一使用以下响应格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 10.2 统一错误码

- 200: 操作成功
- 400: 请求参数错误或业务逻辑错误
- 401: 未登录或token过期
- 403: 无权限（如非管理员访问管理员接口）
- 404: 资源不存在
- 500: 服务器内部错误

### 10.3 认证方式

所有需要登录的接口（除登录接口外）需要在请求头中携带JWT token：

```
Authorization: Bearer {token}
```

### 10.4 时间格式

- 日期格式：`YYYY-MM-DD`
- 时间格式：`HH:mm:ss`
- 日期时间格式：`YYYY-MM-DD HH:mm:ss`

### 10.5 分页说明

分页查询接口统一使用以下参数：

- `page`: 页码，从1开始
- `size`: 每页数量

分页响应格式：

```json
{
  "total": 100,
  "page": 1,
  "size": 10,
  "list": []
}
```
